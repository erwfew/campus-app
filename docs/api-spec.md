> 版本：v1.0  
> 日期：2026年4月  
> 联系人：吕嘉成  
> 说明：本文档用于向学校信息中心说明效园通平台所需的数据接口

---

## 一、对接方式

推荐 **OAuth2.0 授权登录 + REST API** 方式：

```
学生打开效园通 → 跳转学校统一认证 → 授权后返回 token → 凭 token 调用接口
```

- 学校提供统一认证地址（CAS/OAuth）
- 效园通获取 token 后调用学校开放接口
- 所有接口走 HTTPS，保障数据安全

    "grade": "2023",

## 三、通用规范

### 请求格式
- 协议：HTTPS
- 基础路径：`https://api.xxx.edu.cn/campus`
- 认证：`Authorization: Bearer <token>`
- Content-Type：`application/json`

### 响应格式
```json
{
  "code": 0,        // 0=成功, 非0=失败
  "message": "ok",
  "data": { ... }
}
```

### 错误码
| code | 说明 |
|------|------|
| 0 | 成功 |
| 1001 | 未登录/Token 过期 |
| 1002 | 无权限 |
| 2001 | 参数错误 |
| 3001 | 学期不存在 |
| 9999 | 服务器内部错误 |

### 频率限制
- 建议：每用户每分钟不超过 60 次请求
- 课表/成绩等静态数据建议加缓存，有效期 1 小时

## 四、数据安全承诺

效园通承诺：

1. **数据不存储**：课表、成绩等数据仅缓存在用户手机本地，不上传到效园通服务器
2. **HTTPS 传输**：所有接口走 HTTPS 加密传输
3. **Token 隔离**：每个用户独立 token，不共享账号
4. **不转卖数据**：绝不将学校数据用于任何商业用途
5. **接受审计**：配合学校信息中心的安全审查
